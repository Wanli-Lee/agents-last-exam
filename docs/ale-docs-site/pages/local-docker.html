Local Docker

Run the Linux subset in a container on your own machine — no GCP, no VM.

How it works

The docker provider boots agentslastexam/ale-ubuntu22-docker (pulled from Docker Hub on first use), a container whose userspace is an export of the ale-ubuntu22 GCE VM's root filesystem — the same node, Python venv, cua-server and pre-installed task software, at the same paths. Per run unit it starts one container, brings up cua-server on port 5000 behind a virtual X display (Xvfb :0), and drives it over the same cua HTTP API as a cloud VM.

The published image is data-less: it ships no task data and no answers. The data is supplied from your host by the local:<dir> source — fetched once from Hugging Face, then docker cp'd into each container (input before the agent, reference only at grading; see Task spec & data staging).

Run it

Two things to fetch, both one-time; then run as usual.

  1. Image — nothing to do: the provider docker pulls it on first use (~105 GB, ~42 GB compressed).
  2. Task data — request access to the gated Hugging Face dataset, huggingface-cli login, then: 
    scripts/fetch_task_data.sh        # -> ./task-data

The shipped configs/environments/docker.yaml already points at that directory, so you only configure a run against it: 

snapshots:
  cpu-free-ubuntu:
    provider: docker
    image: ale-ubuntu22-docker     # Hub ref + cua port come from the Image entry
    docker:
      shm_size: 2g
      resolution: [1024, 768]
task_data_source: local:task-data
output_path: local

Per-task CPU/memory are derived from the task card's GCE machine type; override with cpus/memory under docker: as a host cap.

Point the run's task list at selected_tasks/linux_no_dind.txt — the 101 Linux tasks this image supports. The 4 tasks that need Docker-in-Docker are excluded (see below).

Unsupported: tasks that need Docker inside the sandbox

Four tasks have their agent (and grader) run docker inside the sandbox — Docker-in-Docker, which this provider does not enable, so an agent cannot complete them here. They are excluded from the Docker task list:

  • engineering/openroad_sky130_ibex_pnr_signoff — runs the pinned openroad/orfs flow via docker run.
  • computing_math/k8s_migration_1 — minikube start --driver=docker + kubectl.
  • business_finance/bpmn_category_governance_restructuring_l3 — docker compose brings up the Flowable stack.
  • business_finance/bpmn_supply_disruption_l3 — docker compose brings up the Flowable stack.
